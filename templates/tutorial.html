Paso 1: Introducción

¿Qué es esta aplicación?

Esta es una aplicación web construida con Flask (framework de Python) que utiliza Firebase Realtime Database para almacenar y gestionar datos de usuarios.

Conceptos clave:
  • Flask: Framework web ligero de Python
  • Firebase: Plataforma de Google para bases de datos en tiempo real
  • CRUD: Create, Read, Update, Delete (operaciones básicas de datos)

Paso 2: Rutas de Flask

¿Qué son las rutas?

Las rutas definen las URLs que los usuarios pueden visitar en tu aplicación. Cada ruta está asociada a una función que se ejecuta cuando se accede a esa URL.

Ejemplo de ruta:
@app.route('/')
def index():
    return render_template('index.html')
Rutas en esta aplicación:
  • / - Página principal (lista de usuarios)
  • /add_user - Añade un nuevo usuario (POST)
  • /delete_user/<id> - Elimina un usuario (POST)
  • /tutorial - Esta página de tutorial
  • /practice - Página de ejercicios

Paso 3: Integración con Firebase

¿Cómo funciona Firebase?

Firebase Realtime Database es una base de datos NoSQL que almacena datos en formato JSON y se actualiza en tiempo real.

Estructura de datos en Firebase:
{
  "users": {
    "user1": {
      "name": "Juan",
      "email": "juan@example.com"
    },
    "user2": {
      "name": "María",
      "email": "maria@example.com"
    }
  }
}
Operaciones principales:
Crear (Create)
ref.child('users').push().set({...})
Leer (Read)
ref.child('users').get()
Actualizar (Update)
ref.child('users').child(id).update({...})
Eliminar (Delete)
ref.child('users').child(id).delete()

Paso 4: Templates HTML

¿Qué son los templates?

Los templates son archivos HTML que Flask puede renderizar dinámicamente usando el motor de plantillas Jinja2.

Sintaxis Jinja2:
{% for user in users %} - Bucle for
{{ user.name }} - Mostrar variable
{% if condition %} - Condicional
Ejemplo práctico:
<table>
  {% for user in users %}
  <tr>
    <td>{{ user.name }}</td>
    <td>{{ user.email }}</td>
  </tr>
  {% endfor %}
</table>
Importante:

Los templates deben estar en la carpeta templates/ para que Flask los encuentre automáticamente.

Paso 5: Flujo Completo

¿Cómo funciona todo junto?

Veamos el flujo completo cuando un usuario añade un nuevo registro:

Usuario llena el formulario

El usuario ingresa nombre y email en el formulario HTML.

Formulario envía POST a /add_user

El navegador envía los datos al servidor Flask mediante HTTP POST.

Flask procesa la petición

La función add_user() recibe los datos y los guarda en Firebase.

Firebase guarda los datos

Los datos se almacenan en la base de datos en tiempo real.

Redirección a página principal

El usuario es redirigido a / donde ve la lista actualizada.
